# Travel Agent 项目 5 学时学习路线

这份路线按“从简单到困难”的顺序安排，目标不是一下子学完所有源码，而是让你在 5 个学时里逐步看懂项目、跑起来、理解核心逻辑，最后具备自己修改和扩展功能的能力。

---

## 第 1 学时：先搞清楚项目是做什么的

### 目标
- 看懂项目的整体目的。
- 知道前端、后端、AI 规划、记忆库分别负责什么。
- 能把项目的主流程用一句话讲出来。

### 重点阅读
- `README.md`
- `架构说明.md`
- `backend/app/api/main.py`
- `frontend/src/App.vue`

### 你要理解的事
- 这是一个“前后端分离”的项目。
- 前端负责收集用户旅行需求和展示结果。
- 后端负责生成旅行计划。
- AI Agent 是核心，它会调用知识库、地图、天气等能力。

### 学完后你应该能回答
- 这个项目解决什么问题？
- 用户从哪里输入需求？
- 结果最后展示到哪里？
- 后端为什么不仅仅是普通 API，而是带有 Agent 流程？

### 小练习
- 用自己的话写 3 句话概括项目。
- 画一个最简单的数据流：`首页表单 -> 后端 -> 行程结果页`。

---

## 第 2 学时：先看前端，理解用户怎么“提交需求”

### 目标
- 看懂页面结构。
- 搞清楚前端怎么收集表单数据。
- 明白前端是怎么请求后端接口的。

### 重点阅读
- `frontend/src/views/Home.vue`
- `frontend/src/services/api.ts`
- `frontend/src/types/index.ts`
- `frontend/src/main.ts`

### 你要理解的事
- `Home.vue` 是首页表单，用户在这里填写城市、日期、偏好、预算等信息。
- `api.ts` 是前端统一请求后端的地方。
- `types/index.ts` 是前后端数据字段的“约定书”。
- `main.ts` 负责挂载路由和 UI 框架。

### 你要关注的关键点
- 表单提交时，数据会变成 `TripFormData` 之类的对象。
- 前端通过 `generateTripPlan()` 调后端 `/api/trip/plan`。
- 前端收到结果后，跳到结果页展示。

### 小练习
- 找出首页里用户必须填写的字段。
- 找出前端请求后端的真实地址。
- 尝试说清楚“前端发出去的数据长什么样”。

---

## 第 3 学时：看后端接口，理解数据怎么进入系统

### 目标
- 看懂 API 是怎么接收请求的。
- 理解请求数据和响应数据的结构。
- 知道配置文件、路由、模型分别放在哪里。

### 重点阅读
- `backend/run.py`
- `backend/app/api/main.py`
- `backend/app/api/routes/trip.py`
- `backend/app/api/routes/poi.py`
- `backend/app/models/schemas.py`
- `backend/app/config.py`

### 你要理解的事
- `run.py` 是后端启动入口。
- `main.py` 注册了 FastAPI、CORS 和路由。
- `trip.py` 是旅行规划相关接口的入口。
- `schemas.py` 定义了 `TripRequest`、`TripPlan`、`DayPlan` 等结构。
- `config.py` 管理环境变量，比如 LLM、地图、图片服务的 key。

### 你要关注的关键点
- `/api/trip/plan` 是生成行程的主要接口。
- `/api/trip/refine` 是多轮精修接口。
- `/api/trip/feedback` 会把用户反馈存起来。
- `/api/poi/photo` 会给景点配图。

### 小练习
- 画出一次请求的路径：`前端表单 -> /api/trip/plan -> 后端返回 TripPlan`。
- 看懂 `TripRequest` 里有哪些字段。
- 记住 `schemas.py` 的作用：统一数据格式。

---

## 第 4 学时：看 AI 核心，理解“它为什么能规划旅行”

### 目标
- 看懂 Agent 的工作方式。
- 知道 LangGraph 的节点是怎么串起来的。
- 明白知识库、记忆库、工具调用分别干什么。

### 重点阅读
- `backend/app/agents/test.py`
- `backend/app/memory/qdrant_manager.py`
- `backend/app/memory/episodic_manager.py`
- `backend/app/memory/init_db.py`
- `backend/app/services/unsplash_service.py`

### 你要理解的事
- `test.py` 里面其实是项目真正的“大脑”。
- 它先检索本地攻略知识，再选景点、餐厅、酒店。
- 它会调用 MCP 工具拿天气、地理位置等数据。
- 它会做质量检查，不合格时可能重写规划。
- `qdrant_manager.py` 管理本地知识库检索。
- `episodic_manager.py` 记录用户反馈，形成长期记忆。
- `unsplash_service.py` 负责给景点找图片。

### 你要关注的关键点
- 什么是 `StateGraph`。
- 各个节点的职责：
  - `rag_expert`
  - `poi_selector`
  - `gather_information`
  - `master_planner`
  - `qa_audit`
  - `refine_agent`
- 为什么项目不是“一个接口直接生成结果”，而是分多个节点协作。

### 小练习
- 用一句话解释 `RAG` 是什么。
- 用一句话解释“episodic memory”是什么。
- 试着说出每个节点在做什么。

## 第 5 学时：串起来看完整流程，并学会改项目

### 目标
- 能把前后端和 Agent 串成完整闭环。
- 知道项目的重点功能都在哪里。
- 具备最基本的修改和扩展能力。

### 重点阅读
- `frontend/src/views/Result.vue`
- `frontend/src/views/TripFeedback.vue`
- `backend/app/api/routes/trip.py`
- `backend/app/agents/test.py`

### 你要理解的事
- 结果页负责展示整个行程、预算、天气、评分、地图。
- 用户可以在结果页直接精修行程。
- 用户反馈会进入后端记忆系统，影响下次规划。
- 这个项目的重点不只是“生成一次行程”，而是“生成 -> 展示 -> 修改 -> 反馈 -> 记住用户偏好”。

### 你要掌握的项目重点
- 前端表单和页面路由。
- 后端 API 设计。
- 数据模型定义。
- Agent 多节点流程。
- RAG 知识库。
- 用户历史记忆。
- 第三方工具调用。
- 结果页的交互和二次编辑。

### 小练习
- 尝试修改一个页面文案。
- 尝试新增一个表单字段，并想清楚它要传到后端哪里。
- 尝试在结果页里找到“行程精修”的入口。

---

## 建议的学习顺序

1. 先看文档，知道项目做什么。
2. 再看前端，知道用户怎么用。
3. 再看后端接口，知道数据怎么流动。
4. 然后看 Agent，知道结果怎么生成。
5. 最后看结果页和反馈机制，知道项目怎么形成闭环。

---

## 学完后你应该具备的能力

- 能独立跑起这个项目。
- 能说清楚每个文件夹的作用。
- 能看懂一次旅行规划请求的完整链路。
- 能定位“表单改哪里”“接口改哪里”“规划逻辑改哪里”。
- 能在不打乱整体结构的前提下增加新功能。
